## NFR1: Scalability
The system must handle sudden spikes in usage during disasters, potentially
serving millions of users concurrently.

## NFR2: Availability
The system must remain operational 24/7, with no single point of failure.

## NFR3: Fault Tolerance
Failure of individual services must not bring down the entire system.

## NFR4: Performance
Emergency reports and alerts must be processed within seconds.

## NFR5: Security & Privacy
Sensitive personal and location data must be protected and accessible only
to authorized entities.

## NFR6: Resilience to Network Failure
The system must function in environments with unstable or intermittent
network connectivity.

## NFR7: Maintainability
The architecture must allow independent updates and scaling of components.
